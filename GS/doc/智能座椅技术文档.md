# 双板通信控制电机旋转座椅项目设计说明书

> 项目类型：STM32 双板串口同步 + 电机驱动控制 + 堵转保护
> 

---

## 1. 项目概述

### 1.1 项目背景
本项目用于实现“电机驱动座椅旋转”的双板协同控制：
- 主板负责人机输入（按键）与动作指令下发；
- 从板接收主板动作命令并同步执行；
- 两块板均具备独立堵转检测能力，任一板检测到异常后可广播急停，实现双板联锁保护。

### 1.2 项目目标
- 实现电机四步状态循环：正转 → 停止 → 反转 → 停止；
- 实现主从板动作同步；
- 实现堵转异常触发的双板紧急停机；
- 保证系统具备基础容错（低速连续判定+回退计数机制）。

### 1.3 功能摘要
1. **主板按键控制状态机**（KEY2 触发状态切换）；
2. **串口2通信协议**（帧头+命令+参数+校验）；
3. **电机方向与PWM控制**（TB6612 驱动）；
4. **编码器测速与堵转检测**（100ms 周期任务）；
5. **双板急停联动**（CMD_EMERGENCY）。
6. **PID 速度闭环**（根据目标转速自动调节 PWM）。

---

## 2. 系统总体方案

### 2.1 系统组成
- 控制MCU：STM32F103
- 电机驱动：TB6612
- 速度反馈：增量式编码器（TIM4 编码器模式）
- 双板通信：USART2（PA2/PA3）
- 调试串口：USART1（PA9/PA10，printf）
- 人机输入：KEY2（PC13，仅主板启用）

### 2.2 主从分工与切换方式
通过宏定义切换角色：
- `#define IS_MASTER_BOARD 1`：主板
- `#define IS_MASTER_BOARD 0`：从板

当前代码默认配置为从板（`IS_MASTER_BOARD 0`）。

### 2.3 关键设计思想
- **统一状态机**：主从板都维护电机状态，主板主动切换、从板被动同步；
- **统一安全策略**：任意一侧堵转，立即本地停机并发送急停报文；
- **轻量协议**：4字节固定帧，解析状态机清晰，适用于资源受限MCU。

## 4. 软件架构设计

### 4.1 代码模块划分
- `USER/MiniBalance.c`：主程序、主从逻辑、状态机、堵转检测、通信命令处理
- `MiniBalance_HARDWARE/SYNC_COMM/`：双板串口通信协议封装与接收中断解析
- `MiniBalance_HARDWARE/MOTOR/`：电机GPIO与PWM初始化
- `MiniBalance_HARDWARE/ENCODER/`：编码器模式配置与增量读取
- `MiniBalance_HARDWARE/KEY/`：按键初始化与消抖扫描
- `SYSTEM/usart/`：串口1调试输出

### 4.2 主循环任务节拍
主循环中每 `delay_ms(10)` 执行一次节拍，内部使用 `timestamp` 计数实现 100ms 周期任务：
- 读取编码器增量；
- 判断是否堵转；
- 必要时触发急停广播。

### 4.3 电机状态机
定义状态：
- `MOTOR_STATE_STOP1`
- `MOTOR_STATE_FORWARD`
- `MOTOR_STATE_STOP2`
- `MOTOR_STATE_BACKWARD`

主板每次按键触发执行：`current_state = (current_state + 1) % 4`。

## 5. 双板通信协议设计

### 5.1 帧格式
固定长度 4 字节：

| 字段 | 长度 | 说明 |
|---|---:|---|
| Header | 1B | 固定 `0xAA` |
| CMD | 1B | 命令字 |
| PARAM | 1B | 参数 |
| CHECKSUM | 1B | `(CMD + PARAM) & 0xFF` |

### 5.2 命令字定义
- `CMD_MOTOR_CTRL = 0x01`：电机动作控制
- `CMD_EMERGENCY = 0x0E`：急停报警（双向）

参数定义（`CMD_MOTOR_CTRL`）：
- `PARAM_STOP = 0x00`
- `PARAM_FORWARD = 0x01`
- `PARAM_BACKWARD = 0x02`

### 5.3 接收解析状态机
串口2中断解析流程：
1. 等待帧头 `0xAA`
2. 读取命令字 `CMD`
3. 读取参数 `PARAM`
4. 读取校验并验证
5. 成功后回调 `Process_Sync_Command(cmd, param)`

若校验失败或异常状态，复位到“等待帧头”。

## 6. 控制逻辑与安全策略

### 6.1 主板控制逻辑
主板在按键触发时：
- 切换本地电机状态并执行对应IO/PWM输出；
- 同步发送 `CMD_MOTOR_CTRL` 给从板；
- 从板接收后按参数执行同样方向/停止动作。

### 6.2 堵转检测逻辑
仅在电机运行状态（正转/反转）下检测：
- 每100ms读取一次编码器增量；
- 速度绝对值 `< 30` 时 `stall_counter++`；
- 否则 `stall_counter--`（不小于0，作为容错回退）；
- 当 `stall_counter >= 3` 判定堵转（约300ms持续低速）。

### 6.3 异常联锁动作
发生堵转时执行：
1. 本板立即停机（BIN1=0, BIN2=0, PWMB=0）；
2. 发送 `CMD_EMERGENCY` 给另一块板；
3. 对端收到 `CMD_EMERGENCY` 后同样立即停机。

此机制确保双板任一异常都可触发全系统安全停止。

### 6.4 PID 速度闭环控制

为提升双电机转速一致性，仅发送固定 PWM（如 `PWMB=2000`）通常不足以保证同步。需在每块板本地引入编码器反馈闭环：
- 设定目标速度 `target_speed`（例如每100ms编码器增量目标值）；
- 实时采样当前速度 `current_speed = abs(Read_Encoder(4))`；
- 计算误差 `error = target_speed - current_speed`；
- 通过 PID 输出增量调节 PWM，形成“慢则加、大则减”的自适应控制。

#### 6.4.1 PID 控制规律
- 当 `current_speed < target_speed`，`error > 0`，PID 输出为正，PWM 增大；
- 当 `current_speed > target_speed`，`error < 0`，PID 输出为负，PWM 减小。

#### 6.4.2 公式
```text
PWM_Output = Kp * error + Ki * error_sum + Kd * (error - last_error)
```

其中：
- `Kp`：比例系数，决定响应快慢；
- `Ki`：积分系数，消除稳态误差；
- `Kd`：微分系数，抑制超调和振荡。

```

#### 6.4.3 与现有系统结合方式
1. 电机进入运行状态（正转/反转）后，启用 PID 闭环更新 `PWMB`；
2. 停止状态时清零积分项，避免再次启动时突变；
3. 堵转或急停触发时，立即将 `PWMB=0` 并冻结 PID 状态；
4. 双板可先分别本地闭环，再通过上层同步策略做速度一致性补偿。

## 7. 核心流程图

### 7.1 主板流程
```text
上电初始化
   ↓
等待按键(KEY2)
   ↓(按下)
状态机切换(4步循环)
   ↓
本地执行电机动作
   ↓
发送同步命令到从板
   ↓
100ms测速与堵转检测
   ↓
若堵转 -> 本地停机 + 发送急停
```

### 7.2 从板流程
```text
上电初始化
   ↓
等待串口同步命令
   ↓
接收CMD_MOTOR_CTRL并执行
   ↓
100ms测速与堵转检测
   ↓
若堵转 -> 本地停机 + 发送急停
   ↓
接收CMD_EMERGENCY -> 立即停机
```

## 9. 已知限制与优化建议

### 9.1 当前限制
1. 通信协议较简化，仅有帧头+和校验，抗干扰能力一般；
2. 堵转判定阈值为固定值（`30`），未随工况自适应；
3. 主从角色通过编译宏切换，不支持运行时动态协商。
4. 目前工程主代码尚未实装 PID 参数整定流程与运行态切换逻辑（本文档已给出集成方案）。

### 9.2 优化方向
1. 增加序号、设备ID、CRC8/CRC16，提升可靠性；
2. 增加通信超时机制（主从心跳+失联停机）；
3. 将速度阈值改为“目标转速比例阈值+滤波”；
4. 增加软启动/软停止（PWM斜坡）提升座椅动作平顺性；
5. 增加故障码上报，支持维护诊断。
6. 增加 PID 在线整定接口（串口命令动态改 `Kp/Ki/Kd`），缩短实车调参周期。

## 10. 交付与使用说明

### 10.1 编译前配置
- 主板固件：将 `IS_MASTER_BOARD` 设为 `1` 编译下载；
- 从板固件：将 `IS_MASTER_BOARD` 设为 `0` 编译下载。

### 10.2 运行检查清单
- [ ] 两板地线共地；
- [ ] USART2 TX/RX 交叉连接正确；
- [ ] 电机驱动供电与逻辑供电稳定；
- [ ] 编码器线序正确，速度反馈有变化；
- [ ] 按键触发后主从动作一致。

### 10.3 结论
该方案已具备“主从同步 + 双向急停联锁 + 堵转保护”的完整闭环，可作为座椅旋转控制项目的可用基础版本。后续可在通信鲁棒性、速度闭环与故障诊断方面持续迭代。
